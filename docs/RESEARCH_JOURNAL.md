# Research Journal

研究日志是工作台中记录人工观察、判断和复盘理由的附录层。它把一个
研究结论与当时可见的行情快照、策略候选和记录人绑定起来，方便日后复核。
它不是策略管线、模拟账本或券商接口，也不会因为写入一条日志而产生交易
授权。

## 权限边界

```text
已完成的行情快照 + 当前策略元数据
                 |
                 v
       人工填写的研究日志草稿
                 |
                 v
      服务器绑定所有者和记录人
                 |
                 v
       不可变的本地研究记录
                 |
                 X  不创建订单、不改策略、不改模拟账户、
                    不改变券商权限或实盘门禁
```

日志的 `authority` 固定为 `research_only`。以下操作永远不由研究日志
触发：

- 修改 `config/default.json`、策略实验室候选或活动策略；
- 写入、重放或清空模拟账户、影子账户、券商订单和成交账本；
- 创建订单意图、调用券商适配器、消费人工批次批准或解除 kill switch；
- 把一条人工观点计入样本外验证、沙箱对账或实盘就绪门槛。

`WATCH`、`HOLD`、`CONSIDER_REDUCE` 等文字只是研究记录中的判断，不是
买卖指令，也不代表系统认可该判断。

## 在工作台中使用

完成启动和登录后，打开 **研究** 页面即可看到“研究日志”区域。一次
记录包含：

- `研究日期`：这条判断对应的收盘研究日，使用 `YYYY-MM-DD`；
- `记录类型`：观察、判断、交易复盘、风险、策略或周复盘；
- `关联证券`：一个已配置的证券，留空表示组合或全市场；
- `标题` 和 `笔记`：人工写下事实、假设、反例和后续检查；
- `研究观点`：未记录观点、继续观察、研究增配、维持研究判断、研究减配或
  暂不纳入；
- `确信度`：选择观点后填写 0 至 100 的整数百分比。

记录人和登录用户由服务器会话补入，浏览器不能在请求体中指定其他
所有者、伪造记录人或添加 `execution_authorized` 等字段。提交成功后页面
会显示新记录的编号，并在时间线中按研究周分组。

### 修正记录

日志没有编辑或删除操作。发现错误时，在原记录上选择 **追加修正**，填写
一条新记录；新记录的 `correction_of` 指向原编号，旧记录保持原样。这样既
能看到当时的判断，也能看到何时、为什么改正。直接改写 JSON 文件会使内容
指纹校验失败，读取时安全失败，不应作为正常修正流程。

### 筛选与状态

页面支持按类型、证券、关键词和返回条数筛选。筛选只影响当前显示，不改变
记录。以下状态有不同含义：

- **空数据**：当前用户尚未写入记录，或筛选条件没有匹配项；
- **正在写入**：提交请求尚未完成，按钮会暂时锁定，避免重复写入；
- **不可用/失败**：目录、文件锁或记录校验失败，页面会保留明确的恢复提示，
  不会用空白内容冒充“没有记录”。

时间线的周标题来自 `research_date` 对应周一的 `week_start`。研究日志本身仍
是人工追加的证据；它不会因为被分组就变成自动日报、周报或交易信号。

## 收盘归档投影

研究页的“收盘归档”是一个按请求即时计算的只读投影，用于把同一个模拟账期
的几类证据放在一起复核。它读取：

| 来源 | 读取内容 | 约束 |
|---|---|---|
| `state/paper_equity.csv` | 逐交易日权益、现金、回撤、日收益、持仓、目标权重、配置指纹和行情快照 ID | 只接受当前 `account_id`，日期必须严格递增，拒绝重复会话、非法数字、跨配置账期和损坏 JSON |
| `reports/paper_YYYYMMDD.json` | 模拟日报的权益、现金、持仓、目标权重、回撤、收益、成交、拒单和原因 | 文件名、日期和账户必须一致；未知字段、重复键、损坏或跨账期文件会显示为错误，不会被当作空数据 |
| 当前登录用户的研究日志 | 对应日期的条目数量、分类、观点和条目指纹 | 通过所有者目录读取；不会读取其他用户的日志，也不会因为归档而写入日志 |

归档以证据日期为行，生成逐日摘要、按 ISO 周一至周日聚合的周度复盘，以及
最近日期的持仓数量快照。快照的估值口径固定为“账本数量”；没有历史行情时
不会回填价格、市值或权重，也不会用零值掩盖缺失。

周度收益按该周已绑定净值账本行的 `daily_return` 复合计算，因此周首交易日的
收益不会被漏掉；它不把只有日报、没有净值账本的记录算进交易日或成交统计。
覆盖不完整或包含未绑定/不一致证据时，周度状态保持 `partial`。
交易日历可用时，归档也会分别列出缺少的交易日和账本中多出的非交易日；
任一情况都保持 `partial`，不会用 `6 / 5` 之类的异常覆盖冒充完整周。

### 状态含义

逐日记录可能出现以下状态：

| 状态 | 含义 |
|---|---|
| `current` | 净值账本和同日模拟日报的关键字段一致 |
| `missing_report` | 有净值账本行，但找不到同日模拟日报 |
| `unbound_report` | 有日报，但没有可绑定到当前账期的净值账本行 |
| `evidence_mismatch` | 两个来源都存在，但权益、现金、持仓、目标权重、回撤、日收益或快照 ID 不一致 |
| `journal_only` | 只有研究日志，没有该日的模拟账本证据 |

投影整体状态为 `current`、`partial`、`empty` 或 `unavailable`。`partial`
表示至少一行或一个来源需要复核；`unavailable` 表示账户或关键输入无法安全
读取。错误会带有恢复动作，服务端和页面不会把失败伪装成“没有数据”。周度
记录还会在交易日历可用时披露预期与实际覆盖日，缺日时标记为 `partial`。

### 查询接口与边界

浏览器会自动调用以下只读接口；集成工具也可以使用同样的有界参数：

```text
GET /api/research/archive
    ?kind=all|daily|weekly
    &date=YYYY-MM-DD
    &week=YYYY-MM-DD       # 必须是 ISO 周一
    &limit=1..52
```

`date` 与 `week` 不能同时出现。响应包含 `schema_version`、查询过滤器、来源
计数、逐日/周度记录、快照、每项来源指纹以及固定的 `authority`。请求只读本地
文件和当前用户日志：不会刷新行情、重跑策略、写入 `reports/` 或 `state/`、
调用券商、改变权限或创建订单。

投影自身仍然不写文件；页面刷新会从原始日报、净值账本和日志重新计算，这些
原始文件始终是权威来源。独立的 `archive-generate` 命令和研究页生成操作可以把
投影结果追加到 `state/research_digests/`：相同来源证据会幂等复用，变化证据会
新增带 `supersedes` 与父指纹的不可变 revision。该持久化层按登录所有者和模拟
账户账期隔离，仍固定为 `research_only`，不会使投影或日志获得交易权限。完整
命令、18:30 全用户计划任务、接口、容量和恢复边界见
[日报/周报归档](RESEARCH_DIGESTS.md)。

持久化 digest、对应 HTTP 接口和计划任务已包含在 `v0.13.0` wheel 中。不带
周期过滤器的单次生成最多处理最近 52 个
日报和最近 52 个周报；更早周期必须分别用 `--date` 或 `--week` 单期回填。
18:10、18:20 和 18:30 是三个独立任务的错峰时间，不保证前序任务已经完成；
`scheduled` 也只是本地操作者可提供的审计标签和 runner 约定，不是认证来源。

## 记录内容和证据

每条记录保存一个可重算的 SHA-256 `entry_fingerprint`，并包含两类证据：

| 证据 | 可用时记录 | 不可用时记录 |
|---|---|---|
| 行情快照 | 共同完成交易日和快照指纹 | `available=false`，日期和指纹均为空 |
| 策略状态 | 活动候选（可选）、策略指纹和生命周期状态 | `available=false`，候选、指纹和状态均为空 |

证据是“当时看到什么”的证明，不是对行情正确性或策略收益的保证。快照
来源、回退原因和完整行情质量仍应在 **数据** 和 **行情** 页面复核；策略
候选的验证、批准和生命周期仍应在策略实验室复核。记录中的证据不可替代
这些独立审计。

## 本地存储和完整性

默认根目录为 `state/research_journal`。如需分离不同工作区，可以在配置中把它改为
工作区 `state/` 目录下的其他子目录；系统会拒绝空路径、仓库外路径和 `state/`
本身，确保日志仍受 Git 忽略规则保护：

```json
{
  "research_journal": {
    "root_dir": "state/research_journal"
  }
}
```

每个登录所有者使用规范化身份的 SHA-256 目录，不在路径中保存明文用户名：

```text
state/research_journal/
└── users/<owner-sha256>/
    ├── .owner.lock
    └── entries/
        └── journal_<32-hex-id>.json
```

写入在进程锁和操作系统文件锁内完成，记录以同目录临时文件落盘后原子创建；
同一编号不能被覆盖。读取会拒绝重复 JSON 键、未知或缺失字段、错误的文件名、
所有者绑定、日期、证据结构或内容指纹。单个所有者最多 2,000 条记录，单条
记录最多 64 KiB；标题最多 80 个字符，笔记最多 4,000 个字符，查询最多返回
200 条。达到容量后需要由操作者在工作区外归档旧数据，再决定是否恢复写入，
应用不会自动删除历史。

## HTTP 接口

浏览器通常不需要直接调用接口。需要集成只读研究工具时，可使用以下有界
接口：

```text
GET  /api/research?category=<category>&symbol=<symbol>&q=<text>&limit=<1..200>
POST /api/research/journal
GET  /api/research/archive?kind=<all|daily|weekly>&date=<YYYY-MM-DD>&week=<YYYY-MM-DD>&limit=<1..52>
GET  /api/research/digests?kind=<all|daily|weekly>&date=<YYYY-MM-DD>&week=<YYYY-MM-DD>&limit=<1..200>&revisions=<0|1>
POST /api/research/digests/generate
```

`GET` 返回研究页数据和 `journal` 对象，包含筛选条件、总数、匹配数、周分组、
记录和固定权限声明。`POST` 只接受 `research_date`、`category`、`symbol`、
`title`、`note`、`decision`、`confidence` 和 `correction_of`；未知字段、
所有者注入和执行授权字段会被拒绝。启用内测登录时，写入需要已认证会话和
会话绑定的 CSRF 令牌；`--owner-local` 只适用于可信的回环地址所有者。

digest 生成请求只接受有界的 `kind`、`date` 与 `week`，服务器继续从会话绑定
所有者和记录人，并把 HTTP 来源固定记录为 `manual`；浏览器提交 `trigger` 会被
拒绝。本机 CLI 可由操作者提供 `manual` 或 `scheduled` 标签，内置任务 runner
约定使用后者；该值不是认证过的 Task Scheduler 来源证明。写入需要 CSRF，容量
冲突返回 HTTP 409。接口错误、损坏记录和目录不可用都会失败关闭。服务端会返回明确
错误，前端显示“不可用”或“失败”状态，不会把错误吞掉后生成一条伪造的研究
记录或日报。

## Git、R2 和备份边界

研究日志是本地运行状态，不是发行包内容，也不属于 GitHub 提交。仓库的
`.gitignore` 通过 `state/*` 忽略整个目录；发布校验也拒绝 `state/` 成员。
不要把日志 JSON、截图中的日志内容或内测导出包提交到公共仓库。

Cloudflare R2 的上传白名单只包含已经校验的 `data/cache/*.csv` 和行情
`manifest.json`。`state/research_journal/` 与 `state/research_digests/`
（以及模拟账户、助理历史、内测用户、券商账本和日志）不会进入 R2 行情
快照。存储页显示的容量和 A/B 类预算因此不包含研究日志或归档对象；R2 连接
失败也不会影响本地日志及归档读写。

当前没有把研究日志或 digest 同步到云端的接口。若需要长期留存，应在停止
工作台后对整个工作区的 `state/research_journal/`、`state/research_digests/`
以及匹配的模拟账期证据做受控、加密的本地备份，并保留原始目录权限；不要把
它们放进 R2 行情命名空间，也不要在备份中混入凭据。恢复后应先让工作台只读
检查当前活动账期的指纹，再继续追加记录。页面、HTTP API 和 CLI 当前没有旧
模拟账期选择器；旧账期 digest 只能与匹配证据一起离线保留，不能通过替换或
手改活动状态来打开，专用的旧账期查看/导出校验器尚未实现。

## 相关实现和验证

- `src/ai_trade/research_journal.py`：记录 schema、指纹、锁、容量和查询；
- `src/ai_trade/research_archive.py`：严格读取账本/日报、按用户合并日志并生成
  只读逐日、周度和持仓数量投影；
- `src/ai_trade/research_digest.py`：按用户和模拟账户账期隔离的日报/周报 revision
  链、来源绑定、幂等追加、容量和完整性校验；
- `src/ai_trade/web/service.py`：绑定行情与策略证据、所有者和记录人；
- `src/ai_trade/web/server.py`：认证、CSRF、研究日志和归档查询边界；
- `tests/test_research_journal.py`、`tests/test_web_research_journal.py`：
  记录不可变性、修正、过滤、损坏检测和 HTTP 输入边界。
- `tests/test_research_archive.py`、`tests/test_web_research_archive.py`：
  账本/日报交叉校验、所有者隔离、损坏和缺失状态、只读保证及接口边界。
- `tests/test_research_digest.py`、`tests/test_web_research_digest.py`：
  幂等复用、`supersedes` 修订链、跨用户/账期隔离、篡改检测、容量和 HTTP 边界。

研究日志的存在不改变 AI Trade 的核心结论：历史收益不代表未来结果，日志也
不构成投资建议或实盘授权。
